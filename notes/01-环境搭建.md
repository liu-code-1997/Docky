# 01 · 环境搭建

本项目本地运行需要三样东西:**Python 环境**、**Ollama(跑模型)**、**Qdrant(存向量,用 Docker 跑)**。

---

## 一、Python 环境

- 要求 **Python 3.11+**。
- ⚠️ **实际踩到的坑**:macOS 自带的 `python3` 可能是旧版本(本机自带 3.9.6),用它建虚拟环境会导致依赖装不上(hatchling 报错)。
  - 解决:用 Homebrew 装的新版本。查找命令:`ls /opt/homebrew/bin/python3.*`,本机有 3.12。
  - 用指定版本建 venv:`python3.12 -m venv .venv`
- 另一个坑:`pyproject.toml` 里项目名是 `rag-docs`,但包目录是 `src/rag`,名字对不上,hatchling 不知道要打包哪个目录。
  - 解决:在 `pyproject.toml` 加上
    ```toml
    [tool.hatch.build.targets.wheel]
    packages = ["src/rag"]
    ```

安装步骤:
```bash
python3.12 -m venv .venv          # 建虚拟环境
source .venv/bin/activate          # 激活(每次开新终端都要)
pip install -e ".[dev]"            # 安装项目依赖(可编辑模式)
```

> `-e`(editable/可编辑模式):改源码立即生效,不用重装。`[dev]` 表示连开发依赖(pytest)一起装。

## 二、Ollama —— 本地模型运行器

### Ollama 是什么(重要概念)

**Ollama 不是"模型",而是"运行模型的工具"。** 类比:Docker 之于容器,Ollama 之于大模型。
- 它负责:下载模型、管理模型、把模型跑起来,并提供一个本地 HTTP 接口供程序调用。
- 真正的"模型"是它跑起来的 `qwen2.5:7b`、`nomic-embed-text` 这些。

### 为什么选 Ollama + 这两个模型

- **省心**:一条命令装、一条命令拉模型、一条命令跑;Mac 上自动用 Metal GPU 加速,无需任何配置。
- **接口形态接近云端 API**:以后切到 OpenAI 等 API 时,代码改动极小(见 [[07-可插拔设计]])。
- **模型选择**(24GB 内存绰绰有余):
  - `qwen2.5:7b`(约 4.7GB)—— 负责**生成答案**的 LLM,中英文都强。
  - `nomic-embed-text`(约 270MB)—— 负责**文本转向量**的 embedding 模型。

### 安装与拉模型

```bash
brew install ollama          # 安装(或从 ollama.com/download 下载)
ollama serve                 # 启动服务(图形版会自动后台运行,可跳过)
ollama pull qwen2.5:7b       # 拉 LLM 模型(约 4.7GB,首次较慢)
ollama pull nomic-embed-text # 拉 embedding 模型(约 270MB)
ollama list                  # 查看已下载的模型
```

### 关键 HTTP 接口(程序就是调这些)

| 接口 | 作用 |
|---|---|
| `GET /api/version` | 查 Ollama 版本(用来确认服务通了) |
| `POST /api/generate` | 给 prompt,生成文本(我们的 LLM 调这个) |
| `POST /api/embeddings` | 给文本,返回向量(我们的 embedder 调这个) |

手动测试:
```bash
curl http://localhost:11434/api/version
curl http://localhost:11434/api/generate -d '{"model":"qwen2.5:7b","prompt":"用一句话说明什么是RAG","stream":false}'
curl http://localhost:11434/api/embeddings -d '{"model":"nomic-embed-text","prompt":"hello"}'
```

## 三、Qdrant —— 向量数据库(Docker 跑)

### Qdrant 是什么

一个专门存"向量"并能快速做"相似度检索"的数据库。我们把文档切块后的向量存进去,提问时来这里找最相近的块。选它是因为它是当前最热门的独立向量库之一,文档质量高。

### 为什么用 Docker 跑

Qdrant 是个独立服务。用 Docker 一条命令就能起,不用在本机装一堆东西,关掉也干净。

```bash
docker compose up -d         # 后台启动 Qdrant(读 docker-compose.yml)
docker ps                    # 确认 rag-qdrant 容器在运行
curl http://localhost:6333/healthz   # 健康检查
```

- 端口 `6333` 是 REST/HTTP API(程序和 curl 用这个)。
- 端口 `6334` 是 gRPC(更快的二进制协议,本项目暂不用)。
- 数据存在 `./qdrant_storage`(已在 .gitignore 忽略)。
- 控制台:浏览器打开 http://localhost:6333/dashboard 可视化查看集合与数据。

## 四、config.py 的意义

所有"模型名、服务地址、切分参数"都集中在 `src/rag/config.py` 一个地方。
好处:以后想换模型、换地址、调切分参数,**只改这一处**,不用满项目找。还能用 `.env` 文件或环境变量覆盖(见 `.env.example`)。
